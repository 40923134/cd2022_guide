20220302 工作流程:

將 cadlab 網路連線接到系 backbone (使用純 IPv6 設定後, 為了連結到 IPv4 網站, 必須透過雙協定支援的 proxy server)

將兩個隨身碟放入電腦 USB 3.0 插槽 (與 2.0 速度差 10 倍)

啟動電腦

在 Windows 10 尚未啟動防毒軟體之前, 以系統管理員身分執行 1_ipv6_network_setup.bat (為了避開蠕蟲惡意的網路攻擊)

批次檔案將會開啟 Firefox 連結到 cd2022 課程倉儲

希望修改 cmsimde 倉儲內容, 以 git clone --recurse-submodules git@github.com:mdecourse/cmsimde.git 取下倉儲資料時, 電腦回報沒有 correct access rights

上個步驟表示 putty session github.com 並沒有設定, 帶出上週在 cdb 從 registry 匯出的 2022_cdb.reg, 直接以滑鼠點擊執行後, 再次執行 git clone --recurse-submodules git@github.com:mdecourse/cmsimde.git 取下 cmsimde 倉儲

但 cmsimde 無法直接產生靜態網頁與動態網頁, 因此擬以 git submodule add 透過另一個 cmsimde_site 倉儲進行修改後, 將資料回應至 cmsimde.

需要新增 github 倉儲, 因此登入 github

因為要取 2a 學員 github 帳號, 以 ssh 登入 17 電腦, 啟動 ethercalc

由於必須從教務主機取得 2a 學員學號, 經由 heroku 主機取得 https://nfulist.herokuapp.com/?semester=1102&courseno=0752&column=True

連接至 cd2022 gitter, 告知擷取 2a github 帳號的 ethercalc 表單連結

cd2022 要求每一學員必須在 github 帳號下建立 cd2022 倉儲與網頁, 並且由各小組組長建立 cd2022ag1-1 (為第一組第一小組), 且每位學員在 cd2022ag1-1 都擁有 git push 權限, 或採 pull requests 修改小組分組倉儲內容.

而每一大組的組長負責建立 cd2022ag1 (為 2a 第一大組) 後, 各大組學員對分組倉儲可以擁有 git push 權限, 或各學員採 pull requests 修改大分組倉儲內容.

各學員在每週會議後, 個人所負責完成的內容或簡報必須先放到個人的 cd2022 倉儲後, 再透過小組協同倉儲練習如何解決組員間提交內容的衝突後, 再整合至各大組的協同倉儲.

接著要從 ethercalc 取下 2a 學員的 github 帳號與大組分組資料 (能不能用程式協助?)

2a w2 assignments:

1. 在個人帳號下, 使用最新版的 cmsimde 建立 cd2022:
git submodule add https://github.com/mdecourse/cmsimde.git cmsimde

或者 git clone --recurse-submodules https://github.com/mdecourse/cmsimde.git

將沒有 git 資料的 cmsimde 目錄放到個人倉儲.

2. 根據 https://mde.tw/cd2022_guide/content/Grouping.html 中的 2b 程式執行, 配合 https://mde.tw/studlist/2022spring/2a.txt 修改為能夠直接在頁面顯示各組員的 cd2022 倉儲與網頁連結, 以及各學員協同大分組倉儲與網頁連結 (例如: cd2022ag1).

3. 能否透過 github api 程式 (也可以逐一進入各班組員 github 帳號, 以手動查驗後, 以表格列出結果), 列出 w2 階段各組員帳號下 cd2022 與 cd2022ag1 (以 2a 第一組組員為例) 是否符合陽春的 cmsimde_site 內容.
